# Промт для AI-агента: Разработка MVP «Ресурсный To-Do»

## Роль и контекст
Ты — опытный Full-Stack разработчик и UX/UI-инженер. Твоя задача — с нуля разработать MVP приложения для управления задачами «Ресурсный To-Do». 
Главная цель продукта — помочь людям планировать день исходя из их реального запаса сил, а не абстрактной продуктивности. Приложение должно снижать тревожность, избегать токсичной продуктивности и помогать находить баланс.

**Технический стек:** Выбери наиболее оптимальный и современный стек технологий для реализации веб- или кроссплатформенного приложения самостоятельно.

---

## 1. UX/UI Философия (Критически важно!)
* **Визуальный покой:** Интерфейс должен быть максимально чистым, минималистичным и читабельным. Используй спокойные, пастельные или нейтральные тона.
* **Никакого осуждения:** Категорически запрещено использовать красный цвет для просроченных задач или превышения лимитов. Отсутствие агрессивных уведомлений и пушей.
* **Свобода выбора:** Приложение только информирует пользователя о перегрузке, но никогда не блокирует добавление новых задач.

---

## 2. Функциональные требования (MVP)

Разработку необходимо вести пошагово, реализуя следующие экраны и модули:

### Шаг 1: Базовый список задач (Main Screen)
* Реализуй простой и удобный интерфейс добавления, редактирования и удаления задач.
* Список должен иметь чекбоксы для отметки выполнения.

### Шаг 2: Модуль «Утренняя энергия»
* При открытии приложения в начале нового дня пользователю должен показываться элемент (например, ползунок или поле ввода), где он отвечает на вопрос: «На сколько ты себя чувствуешь сегодня?».
* Диапазон значений: от 1 до 100.
* Это значение сохраняется как **Бюджет Энергии** на текущий день.

### Шаг 3: Вес задач и Индикатор баланса
* При добавлении задачи пользователь должен выбрать её «Вес» (энергозатратность). Сделай 3 градации:
    * Мелкая (вес 5)
    * Средняя (вес 20)
    * Глыба (вес 50)
* Под списком задач (или в верхней части экрана) реализуй **Мягкий индикатор баланса**. 
    * Он должен показывать соотношение: `Сумма весов задач на сегодня / Твой Бюджет Энергии`.
    * Формат отображения: спокойный Progress Bar.
    * **Особая механика:** Если сумма задач превышает бюджет энергии, линия НЕ становится красной. Она меняет стиль (например, на пунктирную) и рядом появляется текстовая подсказка спокойным шрифтом: *"Сегодня плотный график. Позаботься о себе"*.

### Шаг 4: Библиотека Ресурсов («Мои радости»)
* Создай отдельный раздел или модальное окно, куда пользователь может добавлять активности, которые его восстанавливают (например, «попить кофе», «10 минут соцсетей», «прогулка»).
* Дай возможность быстро копировать/добавлять активности из этого раздела в основной список задач на день.
* Такие задачи не тратят энергию (их вес = 0).
* **Слот «Для себя»:** В основном списке задач (на самом верху) должна быть закрепленная визуальная зона (Top Priority), куда предлагается поставить хотя бы одну ресурсную активность из Библиотеки.

## 3. Порядок действий для агента
1.  **Инициализация:** Настрой проект, базовую архитектуру и роутинг.
2.  **Mock Data:** Создай заглушки данных для тестирования интерфейса без бэкенда.
3.  **UI Каркас:** Сверстай основные экраны (Утренний экран, Список задач, Библиотека ресурсов), соблюдая философию визуального покоя.
4.  **Логика:** Напиши логику подсчета энергии, веса задач и изменения состояния индикатора баланса.
5.  **Ревью:** Предоставь мне промежуточный результат для оценки, прежде чем переходить к сложным интеграциям или полировке.
