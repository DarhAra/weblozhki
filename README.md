# Ресурсный To-Do (Resourceful To-Do)

**Ресурсный To-Do** — это веб-приложение для управления задачами, созданное с фокусом на заботу о себе, избегание выгорания и «токсичной продуктивности». Приложение помогает пользователю планировать день исходя из реального запаса сил (энергии), а не абстрактного количества времени.

---

## 🌟 Основная идея и философия

Главная цель продукта — снизить тревожность при планировании.
*   **Визуальный покой:** Интерфейс выполнен в минималистичном стиле, используются светлые и спокойные тона (никаких агрессивных красных цветов для просроченных задач).
*   **Никакого осуждения:** Если задач запланировано больше, чем есть сил, приложение не ругает и не блокирует добавление новых дел, а лишь мягко напоминает заботиться о себе.
*   **Свобода выбора:** Пользователь сам решает, сколько энергии у него сегодня есть, и какой «вес» (сложность) имеет каждая конкретная задача.

---

## 🚀 Основные функции

### 1. Онбординг (Знакомство)
При первом запуске приложение:
*   Предлагает выбрать **виртуального помощника** в виде аватара (человек, котик, собачка или робот). Аватар будет сопровождать пользователя в интерфейсе и давать мягкие подсказки.
*   Спрашивает **Имя**, чтобы более персонализировано здороваться по утрам.
*   Собирает список того, **что пользователь считает отдыхом** (добавляется в «Библиотеку ресурсов»).

### 2. Утренняя энергия
Каждый новый день начинается с вопроса: **«На сколько ты себя чувствуешь сегодня?»** (от 1 до 100%).
Это значение становится **Бюджетом Энергии** на день.

### 3. Планирование дня (Главный экран)
Пользователь добавляет задачи, задавая им «Вес» (энергозатратность):
*   Пустяк (5)
*   Мелкая (10)
*   Средняя (20)
*   Большая (30)
*   Глыба (50)

Над списком задач находится **Индикатор баланса**. Он показывает соотношение: *Сумма весов задач на сегодня / Бюджет Энергии*.
Если график перегружен (задач больше, чем сил), линия меняет стиль, а виртуальный помощник напоминает о необходимости отдыха.

### 4. Библиотека ресурсов («Мои радости»)
Отдельный раздел для хранения активностей, которые не забирают энергию, а наоборот, восстанавливают её (например, «попить кофе», «погладить кота», «15 минут прогулки»).
Их «Вес» равен 0. Пользователь может в один клик перенести радость из Библиотеки в список задач на день (в специальный блок **Приоритет: для себя**).

### 5. На потом
Раздел, куда можно мягко убрать невыполненные задачи «на потом» (снять с них дату). Они не мозолят глаза в списке «на сегодня», но и не теряются. Из раздела «На потом» всегда можно вернуть задачу на текущий или любой другой день.

### 6. Сделано
Отдельный спокойный раздел для завершённых задач. Сюда попадают дела, которые уже закончены и не требуют возвращения в текущий день.

### 7. Обзор конца дня (Review)
Если день закончился, а задачи остались невыполненными, на следующее утро приложение мягко спросит: «Куда их направим?». 
Вы можете:
*   Перенести их на «сегодня» ☀️.
*   Отправить в «На потом» 📦.

### 8. План на неделю
Встроенный календарь (канбан-доска) на 7 дней вперед, куда можно:
*   Добавлять задачи на будущие даты.
*   Перетаскивать задачи между днями с помощью мышки (Drag-and-Drop).

## 🛠 Техническая реализация (Для разработчиков)

Проект представляет собой классическое **Client-Side** веб-приложение (MVP). Бэкенд и базы данных не используются.

*   **Языки:** HTML5, CSS3, Vanilla JavaScript (ES6+).
*   **Хранение данных:** Состояние приложения (созданные задачи, профиль, настройки онбординга, ресурсы) хранится локально в кэше браузера пользователя (`localStorage` по ключу `resourceTodoState`).
*   **Особенности архитектуры:** Одностраничное приложение (SPA). Экраны переключаются путём скрытия/отображения соответствующих `<div>` блоков (с помощью CSS-класса `.hidden`).
*   **Рабочая версия для запуска:** Исходники приложения живут в папке `js/`, а браузер открывает готовый собранный файл `js/app.bundle.js`. Это позволяет надёжно запускать проект обычным открытием `index.html` через `file://`.

### Структура проекта
*   `index.html`: Вся разметка приложения, модальные окна и экраны (включая шаблоны для задач).
*   `style.css`: Внешний вид. Минимализм, кастомные чекбоксы, сетка, анимации (`fadeIn`, `popIn`), адаптивность (Mobile -> Desktop).
*   `js/`: Основной исходный код приложения, разбитый на модули (`state`, `domain`, `ui`, `onboarding`, `utils`).
*   `js/app.bundle.js`: Готовый собранный файл для браузера. Именно его подключает `index.html`.
*   `script.js`: Старый файл, оставленный в проекте только как архивный legacy-код. Рабочая версия приложения его больше не использует.

### Запуск проекта (Installation & Setup)
Установка не требуется. Проект готов к запуску «как есть».
1.  Скачайте или склонируйте папку с файлами.
2.  Откройте файл `index.html` в любом современном браузере.
3.  Для сброса данных (например, чтобы заново пройти онбординг), откройте консоль разработчика (`F12`), перейдите во вкладку `Application` (или `Storage`) -> `Local Storage` -> и удалите ключ `resourceTodoState`. Затем перезагрузите страницу.

### Пересборка готового файла
Если меняется код в папке `js/`, один раз установите зависимости, а затем пересобирайте браузерный файл:

```bash
npm install
npm run build
```
